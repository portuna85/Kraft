# `www.kraft.io.kr` “최고의 번호” 개선 제안서

## 1. 개선 방향 요약

`www.kraft.io.kr`의 “최고의 번호”는 로또 번호 예측 서비스라기보다, 과거 당첨 데이터와 조합 규칙을 활용해 **사람들이 자주 고르는 편향 조합을 피하도록 돕는 번호 생성 도구**로 보는 것이 적절합니다.

현재 가장 큰 장점은 “당첨 확률 예측”이라고 과장하지 않고, **편향 회피 도구**라는 점을 명시한다는 것입니다. 다만 사용자가 이를 명확히 이해하도록 만드는 설명, 데이터 출처, 알고리즘 투명성, 책임 있는 복권 이용 안내가 더 강화될 필요가 있습니다.

핵심 개선 목표는 다음과 같습니다.

1. **예측 서비스가 아니라 편향 회피 도구임을 더 명확히 전달**
2. **당첨 확률은 모든 조합이 동일하다는 점을 반복적으로 안내**
3. **데이터 출처, 갱신 주기, 오류 정정 기준 공개**
4. **추천 알고리즘의 규칙과 한계 투명화**
5. **뉴스·통계·조합 분석 기능의 품질 개선**
6. **개인정보, 보안, 책임 있는 복권 이용 안내 보강**

## 3. 핵심 문제 정의

### 3.1 사용자가 “추천 번호 = 당첨 확률 상승”으로 오해할 수 있음

사이트가 “확률 예측이 아님”을 명시하고 있더라도, “최고의 번호”라는 이름과 “추천” 기능은 사용자에게 고확률 번호처럼 받아들여질 수 있습니다.

**개선 필요점**

- 번호 추천 결과 화면마다 다음 문구를 명확히 표시해야 합니다.

> 이 번호는 당첨 확률을 높이는 번호가 아닙니다. 모든 6개 조합의 1등 확률은 동일합니다. 이 도구는 생일 번호, 연속 번호, 특정 패턴 쏠림 등 사람들이 자주 고를 가능성이 있는 조합을 피하는 데 목적이 있습니다.

### 3.2 과거 출현 빈도 해석 위험

번호별 출현 빈도, 홀짝 분포, 합산 구간 등의 통계는 과거 결과를 설명하는 데는 유용합니다. 그러나 사용자가 이를 미래 당첨 가능성으로 해석하면 오류가 됩니다.

**개선 필요점**

- 모든 통계 페이지에 “과거 빈도는 미래 추첨 확률을 의미하지 않는다”는 안내를 넣어야 합니다.
- “많이 나온 번호”, “적게 나온 번호”라는 표현을 사용할 경우 해석 주의 문구를 함께 제공해야 합니다.

### 3.3 데이터 출처와 갱신 기준 부족

로또 관련 서비스에서는 데이터 정확도가 핵심입니다. 현재는 회차 정보와 당첨번호를 보여주지만, 데이터 출처, 마지막 갱신 시각, 오류 정정 정책이 명확하게 드러나지 않습니다.

**개선 필요점**

- 데이터 출처 명시
- 마지막 갱신 시각 표시
- 갱신 실패 시 상태 표시
- 오류 제보 또는 정정 요청 경로 제공

## 4. 기능별 개선안

## 4.1 번호 추천 기능

### 현재 장점

- 과거 1등과 완전히 동일한 조합 제외
- 6개 모두 31 이하인 생일 편향 조합 제외
- 완전 등차수열 제외
- 5개 이상 연속 번호 제외
- 같은 십의 자리 번호 5개 이상 몰림 제외
- 편향 회피라는 목적 자체는 타당함

### 개선안

#### 4.1.1 추천 결과에 “추천 사유” 표시

각 추천 조합 옆에 왜 이 조합이 생성되었는지 설명해야 합니다.

예시:

```text
추천 번호: 8, 14, 23, 29, 36, 42
추천 사유:
- 31 이하 번호에 과도하게 몰리지 않음
- 5개 이상 연속 번호 없음
- 같은 십의 자리 번호 5개 이상 몰림 없음
- 과거 1등 조합과 완전 일치하지 않음
```

#### 4.1.2 “확률 점수” 대신 “편향 회피 점수” 사용

“당첨 가능성”, “고확률”, “예측 점수” 같은 표현은 피해야 합니다.

권장 표현:

- 편향 회피 점수
- 흔한 조합 회피 점수
- 분산형 조합 점수
- 중복 당첨 리스크 완화 점수

비권장 표현:

- 당첨 확률 점수
- AI 예측 점수
- 1등 가능성
- 고확률 번호

#### 4.1.3 추천 알고리즘 공개

사이트 내에 `/methodology` 또는 `/about-algorithm` 페이지를 만들고 다음 내용을 공개하는 것이 좋습니다.

- 사용 데이터 범위
- 제외 규칙
- 추천 번호 생성 절차
- 무작위 요소 포함 여부
- 과거 당첨번호와 비교 방식
- 추천 번호가 당첨 확률을 높이지 않는 이유

#### 4.1.4 사용자가 규칙을 선택할 수 있게 하기

고급 설정을 제공하면 신뢰도와 사용성이 올라갑니다.

예시 옵션:

- 생일 번호 편향 피하기
- 연속 번호 피하기
- 같은 십의 자리 몰림 피하기
- 홀짝 균형 고려하기
- 합산 범위 균형 고려하기
- 과거 1등 조합 제외하기
- 완전 무작위 생성하기

## 4.2 최신 회차 기능

### 현재 장점

- 최신 회차 당첨번호 표시
- 당첨금, 당첨자 수, 세후 예상 수령액 제공
- 기본 정보 제공 기능은 실용적임

### 개선안

#### 4.2.1 데이터 갱신 상태 표시

최신 회차 페이지에 다음 항목을 추가해야 합니다.

```text
데이터 출처: 동행복권 공식 발표 기준
마지막 갱신: YYYY-MM-DD HH:mm
갱신 상태: 정상 / 지연 / 오류
다음 예상 갱신: YYYY-MM-DD HH:mm
```

#### 4.2.2 판매점 정보 수집 상태 명확화

판매점 정보가 아직 수집되지 않은 경우, 단순히 “없음”으로 보이면 사용자가 데이터 누락인지 실제 미공개인지 구분하기 어렵습니다.

권장 문구:

```text
당첨 판매점 정보는 아직 공식 데이터가 확인되지 않았습니다. 확인 후 자동 반영됩니다.
```

#### 4.2.3 세후 수령액 계산식 공개

세후 예상 수령액은 유용하지만, 계산식이 보이지 않으면 신뢰도가 낮아집니다.

표시 예시:

```text
세금 계산 기준:
- 200만 원 이하: 비과세
- 200만 원 초과 ~ 3억 원 이하: 기타소득세 및 지방소득세 합산 22%
- 3억 원 초과분: 기타소득세 및 지방소득세 합산 33%
```

## 4.3 번호별 출현 빈도 기능

### 현재 장점

- 번호별 과거 출현 횟수를 직관적으로 볼 수 있음
- 데이터 탐색 기능으로는 흥미가 있음

### 개선안

#### 4.3.1 통계 해석 안내 강화

빈도 페이지 상단에 다음 문구를 추가하는 것이 좋습니다.

```text
주의: 출현 빈도는 과거 당첨 결과를 집계한 값입니다. 과거에 많이 나온 번호가 다음 회차에 더 잘 나오는 것은 아니며, 적게 나온 번호가 곧 나올 가능성이 높다는 의미도 아닙니다.
```

#### 4.3.2 기간 필터 추가

전체 회차만 보여주면 최근 경향을 보고 싶은 사용자의 요구를 충족하기 어렵습니다.

권장 필터:

- 전체 회차
- 최근 50회
- 최근 100회
- 최근 300회
- 특정 회차 구간 직접 입력

단, 기간 필터에도 “미래 예측 근거가 아님”을 표시해야 합니다.

#### 4.3.3 시각화 개선

현재 빈도는 단순 수치 중심이라면, 다음 시각화를 추가하면 이해도가 올라갑니다.

- 번호별 막대그래프
- 평균 출현 횟수 기준선
- 최대/최소 번호 강조
- 번호대별 분포
- 최근 구간과 전체 구간 비교

## 4.4 패턴 통계 기능

### 현재 장점

- 홀짝 분포
- 합산 구간
- 번호 패턴 분석
- 사용자가 조합을 이해하는 데 도움을 줌

### 개선안

#### 4.4.1 “정상적으로 자주 나오는 패턴”과 “당첨 가능성이 높은 패턴”을 구분

예를 들어 3홀 3짝이 과거에 자주 나왔다고 해서 다음 회차의 당첨 확률이 높아지는 것은 아닙니다.

권장 설명:

```text
3홀 3짝 조합은 가능한 조합 수 자체가 많기 때문에 과거 결과에서도 자주 관찰됩니다. 이는 다음 회차에서 3홀 3짝이 반드시 유리하다는 의미가 아닙니다.
```

#### 4.4.2 가능한 전체 조합 대비 비율 표시

과거 출현 빈도만 보여주면 사용자가 오해할 수 있습니다. 실제 가능한 전체 조합에서 해당 패턴이 차지하는 비율과 과거 출현 비율을 함께 보여주면 더 정확합니다.

표시 예시:

| 패턴 | 가능한 전체 조합 비율 | 과거 출현 비율 | 해석 |
|---|---:|---:|---|
| 3홀 3짝 | 약 XX% | XX% | 일반적인 분포 |
| 6홀 0짝 | 약 XX% | XX% | 드문 분포 |

## 4.5 조합 분석 기능

### 개선안

사용자가 직접 입력한 번호에 대해 “좋다/나쁘다”식 판단보다 “편향 특성”을 설명하는 방식이 바람직합니다.

권장 출력 예시:

```text
입력 번호: 1, 2, 3, 4, 5, 6
분석 결과:
- 6개 번호가 모두 연속 번호입니다.
- 매우 많은 사용자가 선택할 가능성이 있는 상징적 조합입니다.
- 당첨 확률 자체는 다른 조합과 동일하지만, 실제 당첨 시 공동 당첨자가 많아질 가능성이 있습니다.
```

이렇게 하면 수학적으로 틀리지 않으면서도 서비스 목적을 분명히 전달할 수 있습니다.

## 4.6 번호 궁합 기능

### 문제점

“번호 궁합”이라는 표현은 사용자가 번호 간에 어떤 특별한 상관관계가 있다고 오해할 수 있습니다. 로또 번호는 독립적인 무작위 추첨이므로 특정 번호 조합이 본질적으로 더 잘 맞는다는 식의 표현은 부적절할 수 있습니다.

### 개선안

#### 4.6.1 명칭 변경

권장 명칭:

- 함께 출현한 기록
- 동반 출현 통계
- 번호쌍 과거 빈도
- 번호 조합 이력

비권장 명칭:

- 번호 궁합
- 찰떡 번호
- 강한 번호쌍
- 당첨 궁합

#### 4.6.2 해석 주의 문구 추가

```text
동반 출현 횟수는 과거에 함께 나온 기록일 뿐이며, 다음 추첨에서 함께 나올 가능성이 높다는 뜻은 아닙니다.
```

## 4.7 로또 뉴스 기능

### 현재 문제점

뉴스 페이지는 로또 관련 키워드 기반 수집으로 보이며, 실제 로또 구매자에게 유용하지 않거나 품질이 낮은 글이 섞일 수 있습니다.

### 개선안

#### 4.7.1 뉴스 분류 체계 도입

뉴스를 다음 카테고리로 나누는 것이 좋습니다.

- 공식 회차 결과
- 당첨자/판매점 소식
- 동행복권 공지
- 복권 제도/세금
- 사행성·피해 예방
- 기타 키워드 뉴스

#### 4.7.2 신뢰도 필터 추가

뉴스 출처를 다음 기준으로 나누어 표시합니다.

| 등급 | 기준 |
|---|---|
| 공식 | 동행복권, 정부·공공기관 |
| 언론 | 등록 언론사 |
| 일반 | 블로그, 커뮤니티, 기타 사이트 |

#### 4.7.3 무관 뉴스 제거

“로또”라는 단어가 들어갔지만 복권과 무관한 정치·선거·광고성 글은 제외하는 필터가 필요합니다.

필터링 기준 예시:

- 제목과 본문에 “로또 6/45”, “동행복권”, “당첨번호”, “복권” 등 핵심 키워드가 함께 있는지 확인
- “로또 선거”, “아파트 로또”, “분양 로또” 등 은유적 표현은 별도 분류 또는 제외
- 저품질 도메인 반복 노출 제한

## 5. 신뢰성 개선안

## 5.1 운영자 정보 공개

사이트 하단 또는 소개 페이지에 다음 정보를 제공하는 것이 좋습니다.

- 운영 주체
- 문의 이메일

## 9. 통계 설명 보강안

## 9.1 독립 시행 설명

로또 추첨은 각 회차가 독립적인 무작위 사건입니다. 따라서 지난주에 나온 번호가 이번 주에 덜 나오거나, 오래 안 나온 번호가 이번 주에 더 잘 나오는 것은 아닙니다.

```text
로또 추첨은 매 회차 독립적으로 이루어집니다. 과거 결과는 다음 회차 번호를 예측하는 근거가 되지 않습니다.
```

## 9.2 도박사의 오류 안내

사용자가 자주 빠지는 오류를 설명하면 교육적 가치가 높아집니다.

예시:

```text
“오랫동안 나오지 않은 번호는 곧 나올 것이다”라는 생각은 도박사의 오류에 해당합니다. 무작위 추첨에서는 이전 결과가 다음 결과의 확률을 바꾸지 않습니다.
```

## 9.3 기대값 설명

복권은 오락성 지출임을 설명해야 합니다.

예시:

```text
로또는 판매액 중 일부만 당첨금으로 지급되는 구조이므로, 장기적으로는 평균 기대수익이 구매금액보다 낮습니다. 따라서 수익 목적이 아니라 오락 목적으로 접근하는 것이 적절합니다.
```

## 10. 콘텐츠 개선안

## 10.1 FAQ 추가

추천 FAQ:

### Q1. 이 사이트가 당첨번호를 예측하나요?

아닙니다. 당첨번호를 예측하지 않습니다. 과거 데이터와 조합 규칙을 참고해 흔한 선택 패턴을 피하도록 돕습니다.

### Q2. 추천 번호를 사면 당첨 확률이 올라가나요?

아닙니다. 모든 6개 조합의 1등 확률은 동일합니다.

### Q3. 그러면 왜 추천 번호를 쓰나요?

생일 번호, 연속 번호, 상징적 조합처럼 많은 사람이 고를 가능성이 있는 번호를 피하면, 실제 당첨 시 공동 당첨자가 많아지는 상황을 일부 줄이는 데 도움이 될 수 있습니다.

### Q4. 과거에 많이 나온 번호가 더 유리한가요?

아닙니다. 과거 출현 빈도는 미래 추첨 확률을 의미하지 않습니다.

### Q5. 번호를 여러 번 생성하면 더 좋은 번호가 나오나요?

아닙니다. 여러 번 생성해도 당첨 확률 자체가 개선되지는 않습니다.
